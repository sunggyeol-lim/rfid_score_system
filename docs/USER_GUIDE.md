# No-Coding SDLC 프레임워크 이용 가이드 (User Guide)

본 가이드는 사용자가 AI 코딩 에이전트(Antigravity)와 협업하여 코딩 한 줄 없이(No-Coding) 결과물을 안전하고 검증된 상태로 만들어가는 흐름을 설명합니다.

---

## 1. 프레임워크 시작하기 (초기 세팅)
에이전트가 가장 먼저 개발 환경을 구성하도록 지시합니다.
1. 터미널에서 아래 명령을 구동해 달라고 에이전트에게 지시하거나 직접 실행합니다.
   ```bash
   node scripts/bootstrap.js
   ```
2. 이 스크립트가 실행되면 프로젝트의 기본 `package.json`과 검증에 필요한 도구(`jest` 등)가 설치되고, `src/` 폴더가 자동 생성됩니다.

---

## 2. No-Coding 개발 협업 프로세스 (기능 개발 방법)

### Step 1: 자연어로 요구사항 지시하기
사용자는 만들고 싶은 기능을 자연어로 작성하여 대화창에 보냅니다.
> **예시**: "사용자가 RFID 태그 정보와 점수를 입력하면 이를 저장하고, 전체 랭킹을 보여주는 Node.js 콘솔 프로그램을 만들어줘."

### Step 2: 요구사항 및 아키텍처 승인 (Human-in-the-Loop)
1. 에이전트는 사용자의 입력을 받아 분석한 뒤 `templates/REQUIREMENT.md`와 `templates/ARCHITECTURE_SPEC.md`를 작성합니다.
2. 작성된 기획 내용과 파일 구조를 확인한 후, 에이전트 인터페이스 상의 **Proceed (승인)** 버튼을 눌러 개발 단계를 시작하게 합니다.
3. 수정할 사항이 있다면 승인을 누르지 말고 대화창에 피드백을 주면 에이전트가 문서를 자동 수정합니다.

### Step 3: 에이전트의 자체 개발 및 검증 (Self-Healing)
* 에이전트가 승인을 확인하면 `src/` 내에 소스 코드와 테스트 코드를 함께 구현합니다.
* 에이전트는 자동으로 `node scripts/run_linter.js` 및 `node scripts/run_tests.js`를 돌려 코드를 스스로 완벽히 검증합니다.

### Step 4: 완료 보고서 확인
* 모든 검증이 끝나면 에이전트가 완성 보고서(`walkthrough.md`)와 테스트 리포트(`docs/LATEST_TEST_REPORT.md`)를 정리하여 사용자에게 전달합니다.
* 사용자는 결과물을 실행하여 확인만 하면 됩니다.
