# Antigravity Agent Custom Rules

이 프로젝트(`rfid_score_system`)는 사용자의 **No-Coding** 요구사항을 바탕으로 AI 에이전트가 주도하여 개발, 검증, 디버깅을 모두 완료하는 자동화 프레임워크 체계를 따릅니다. 
이 워크스페이스에서 작업을 시작하는 에이전트는 아래 규칙을 엄격히 준수해야 합니다.

---

## 1. 기본 원칙
1. **사용자 소통**: 모든 대화는 명확하고 간결한 한국어로 진행하며, 전문 용어를 과도하게 쓰기보다 직관적으로 설명합니다.
2. **Human-in-the-Loop**: 코드를 직접 수정하거나 생성하기 전에 반드시 설계/계획 단계를 거쳐 사용자의 **명시적 승인(Proceed)**을 받습니다.

---

## 2. SDLC 단계별 행동 지침

### [단계 1] 요구사항 구체화 (Analysis)
* 사용자가 아이디어나 기능을 주면 바로 코딩하지 말고, `templates/REQUIREMENT.md` 템플릿에 맞추어 기능 정의서 및 사용자 시나리오를 구체적으로 정리하여 문서로 만듭니다.
* 명확하지 않은 정책이 있다면 사용자에게 먼저 질문하여 합의합니다.

### [단계 2] 설계 및 구현 계획 (Planning)
* 수정/생성될 파일 구조, 사용될 Node.js 패키지, 데이터 흐름을 명세화합니다.
* `templates/ARCHITECTURE_SPEC.md`에 이를 기록하고 사용자의 승인을 받습니다.

### [단계 3] 구현 및 테스트 코드 작성 (Implementation)
* 기능 코드와 함께 이를 검증할 단위/통합 테스트 코드(Jest/Vitest 등)를 반드시 세트로 작성합니다.
* 패키지 관리는 `package.json`을 통해 수행하며, 모든 의존성은 명확히 관리되어야 합니다.

### [단계 4] 자가 치유 검증 루프 (Self-Verification & Debug)
* 코드가 작성되면 반드시 `node scripts/run_linter.js` 및 `node scripts/run_tests.js`를 실행하여 정적 분석 및 테스트 결과를 점검합니다.
* **만약 에러가 발생하거나 테스트가 실패할 경우, 사용자의 도움 없이 에러 로그를 직접 분석하여 코드를 수정하는 피드백 루프를 반복 수행합니다.**
* 모든 검사(Linter, Test Runner)가 100% 성공(Pass)할 때까지 다음 단계로 넘어가지 않습니다.

### [단계 5] 최종 확인 및 결과 보고 (Walkthrough)
* 검증이 끝나면 최종 구현된 파일들의 경로와 테스트 스크린샷/데모 방법 등이 담긴 `walkthrough.md`를 업데이트하여 사용자에게 보고합니다.

---

## 3. 기술 스택 제약 조건
* **런타임**: Node.js v24 이상
* **패키지 매니저**: npm (보안 정책 우회를 위해 스크립트 실행 시 필요 시 `-ExecutionPolicy Bypass` 옵션 고려)
* **테스트 도구**: Jest 또는 Vitest (경량화 및 속도를 위해 Jest 선호)
* **코드 스타일**: ESLint 및 Prettier 표준 준수
