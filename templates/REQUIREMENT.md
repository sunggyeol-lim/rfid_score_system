# 요구사항 정의서 (Requirement Spec) - 1단계 (NFC, 모바일-웹 공통 다중 테마)

* **작성 에이전트**: Antigravity
* **최종 업데이트**: 2026-06-23
* **프로젝트 이름**: RFID 실시간 스코어보드 시스템 (RFID Score System)

---

## 1. 개요 (Overview)
* **목표**: 스마트폰 내장 NFC 센서를 활용하여 학생의 RFID 카드/스티커를 스캔하고 점수를 부여하며, 부여된 점수를 실시간으로 웹 대시보드 화면에 시각화하는 시스템 구축.
* **비주얼 컨셉**: 모바일 앱과 웹 대시보드가 **일관된 디자인 테마(꿀벌, 바다, 놀이터)**를 공유하여 일체감 있는 사용자 경험(UX) 제공.

---

## 2. 유저 시나리오 / 스토리 (User Stories)
1. **관리자 (선생님/스태프) 모바일 앱 시나리오**:
   * Android 또는 iOS 스마트폰에서 전용 **NFC 점수 부여 앱**을 실행합니다.
   * 앱 실행 시 현재 활성화된 테마(예: 꿀벌)에 맞는 귀여운 노란색 벌집 테마 UI가 앱 화면 전체에 적용되어 나타납니다.
   * 필요시 앱 내 설정에서 테마(바다, 놀이터 등)를 변경하면, 앱의 색상과 일러스트가 즉시 시원한 파란색 바다나 파스텔톤 놀이터 분위기로 전환됩니다.
   * 원하는 점수를 누르고 학생 태그를 찍어 점수를 부여합니다.
2. **실시간 스크린 대시보드 웹사이트 시나리오**:
   * 대형 화면에 대시보드 웹사이트를 띄워놓습니다. 
   * 관리자가 폰으로 카드를 찍는 순간, 모바일 앱과 **동일한 테마 비주얼 컨셉** 하에 디자인된 대시보드 화면에서 해당 학생의 점수 상승 정보와 테마 마이크로 애니메이션이 실시간으로 표출됩니다.

---

## 3. 기능 요구사항 (Functional Requirements)

### 3.1. 모바일 앱 클라이언트 (Flutter 기반 Android/iOS 공용)
* [ ] **REQ-MOB-001**: Flutter 기반 크로스 플랫폼 빌드 구조 설계 (Android, iOS 지원).
* [ ] **REQ-MOB-002**: 점수 선택 UI (예: +1, +5, +10점 등 직관적인 버튼 배치).
* [ ] **REQ-MOB-003**: 1단계 가상 NFC 태깅 입력 및 시뮬레이션 UI 탑재.
* [ ] **REQ-MOB-004**: 백엔드 REST API 통신 모듈 구현 (선택된 카드 ID와 점수 값을 JSON으로 전송).
* [ ] **REQ-MOB-005**: 2단계 센서 연동을 고려한 NFC 리더 인터페이스 추상화 설계.
* [ ] **REQ-MOB-006**: **다중 비주얼 테마 지원 및 모듈화 (Flutter)**:
  * 웹 대시보드와 동일한 3대 테마(꿀벌, 바다, 놀이터) 색상 셋 및 비주얼 테마 정의.
  * 테마 체인저 모듈을 구현하여, 앱 내 선택에 따라 UI의 배경색, 버튼 스타일, 폰트, 마스코트가 즉각적으로 교체되는 구조 설계.

### 3.2. 다중 테마형 웹 대시보드 (Web Dashboard)
* [ ] **REQ-DSH-001**: 현재 등록된 모든 학생의 랭킹 스코어보드 표출.
* [ ] **REQ-DSH-002**: WebSocket(Socket.io)을 통한 실시간 데이터 수신 및 화면 리렌더링.
* [ ] **REQ-DSH-003**: 점수 부여 이벤트 발생 시 알림 팝업 및 개별 학생 카드 점수 상승 마이크로 애니메이션 연출.
* [ ] **REQ-DSH-004**: **다중 비주얼 테마 연출 및 테마 토글(전환) 인터페이스**:
  * **테마 1: 귀여운 꿀벌 (Honeybee)**: 옐로우/브라운톤 배색, 벌집 모티브 UI, 점수 획득 시 꿀벌 비행 마이크로 애니메이션.
  * **테마 2: 시원한 바다 (Ocean)**: 블루/시안톤 배색, 파도 및 방울 모티브 UI, 점수 획득 시 물방울 상승 및 파도 치는 애니메이션.
  * **테마 3: 아기 친구들의 놀이터 (Kids Playground)**: 파스텔톤 배색, 놀이터/장난감 모티브 UI, 점수 획득 시 풍선이 날아가거나 아기 아이콘이 점프하는 애니메이션.
* [ ] **REQ-DSH-005**: **테마 모듈화 및 확장성**: CSS Variables 및 모듈식 CSS 설계를 통해 신규 테마 추가가 매우 쉽고 기존 코드를 건드리지 않는 구조 제공.

### 3.3. 백엔드 API 서버 (Node.js Express)
* [ ] **REQ-BE-001**: 모바일 앱에서 전송하는 점수 부여 API 엔드포인트 제공.
* [ ] **REQ-BE-002**: WebSocket 서버 구동 및 대시보드 클라이언트로 실시간 이벤트 브로드캐스팅.
* [ ] **REQ-BE-003**: 데이터베이스 영속화 (학생 정보 및 점수 획득 로그).

---

## 4. 비기능 요구사항 (Non-Functional Requirements)
* **디자인 일관성**: 모바일 앱과 웹 대시보드 간의 디자인 아이덴티티(Hex 색상 코드, 마스코트 캐릭터 컨셉)가 100% 일치해야 함.
* **성능**: 테마 전환 시 앱과 대시보드 모두 레이턴시나 프레임 드랍 없이 자연스러운 트랜지션이 일어나야 함.
